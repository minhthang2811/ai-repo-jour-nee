The Problem

Why Agentic Engineering Matters

AI coding tools have supercharged software development, but simple text-generation interfaces lead to severe limitations as projects scale.

😤

Vibe Coding Chaos

Writing code without structured guardrails or test-driven cycles leads to disjointed, bug-ridden implementations that collapse as soon as codebase size and logical complexity increase.

🔧

Context & Token Bloat

Without isolated agent boundaries, session histories quickly balloon with raw file contents and tool outputs, causing context rot. The model enters a "dumb zone" and starts ignoring instructions.

⏱️

Repetitive Prompting

Developers waste significant time feeding Claude the same compilation steps, lint rules, and instructions for every new task. Manual process checking is slow, expensive, and error-prone.

claude-code-best-practice solves this.

By formalizing modular primitives—custom slash commands, lifecycle hooks, progressive-disclosure skills, and isolated subagents—this repository converts Claude Code from a chat tool into a reliable, developer-controlled agentic engineering pipeline.

Architecture

How It Works

A premium, multi-layered system designed to minimize token overhead, protect context, and enforce strict execution standards across all agent operations.

🎯

Slash Commands

Custom entry points defined in .claude/commands/. They initiate clean, reproducible workflows (e.g. /weather-orchestrator, /ce-plan) and handle developer queries directly.

01
02
🤖

Context Isolation (Subagents)

The primary session delegates heavy tasks to isolated subagents. Raw search outputs, greps, and dead ends are contained within the subagent context, returning only the final report to the main session.

🧩

Skills Engine

Progressive disclosure folders containing SKILL.md, references, and scripts. They are automatically lazy-loaded only when the agent touches matching globs, saving precious token window space.

03
04
⚡

Lifecycle Hooks

Event-driven scripts in .claude/hooks/ that trigger on session start, tool use, cwd changes, or stop. Enforces auto-formatting, checks permissions, and plays system sound notifications.

📡

MCP Server Integrations

Model Context Protocol servers that act as API gateways. Connecting Claude Code directly to local filesystems, external databases, or browser automations via standard JSON-RPC interfaces.

05
06
📋

Memory & State Guard

Utilizes a lightweight, strict CLAUDE.md memory file (< 200 lines) combined with global and project rules to enforce constraints, command aliases, and build specifications.

🏗️

The Engineering Loop

Converges all actions onto the systematic Research → Plan → Execute → Review → Ship cycle, backed by automated test-driven development (TDD) gates.

Get Started

How to Use

Follow these steps to learn, practice, and implement Claude Code best practices in your own local environment.

01

Install Claude Code

Ensure Node.js is installed, then globally install the official Anthropic Claude Code CLI.

$ npm install -g @anthropic-ai/claude-code
02

Clone the Best Practice Repository

Clone the codebase containing the commands, hooks, and skills, then enter it.

$ git clone https://github.com/shanraisshan/claude-code-best-practice.git
cd claude-code-best-practice
03

Open the Interactive CLI Session

Launch the Claude Code TUI to load local rules and custom configurations.

$ claude
04

Run the Weather Orchestrator Demo

Once inside the interactive prompt, trigger the weather demo to watch the orchestration workflow execute.

/weather-orchestrator
