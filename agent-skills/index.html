The Problem

AI Agents coding without discipline

😤

Agentic Drift

AI assistants often jump straight to modifying codebase files before fully mapping out design constraints, leading to unstable systems and code regressions.

🔧

Impulsive Edits

Without standard protocols, agents might write overly complex structures, introduce silent security issues, or skip refactoring tasks to complete jobs quickly.

⏱️

Verification Gaps

Coding tools often skip test writing and automated checks. When bugs occur, developers must spend critical time debugging agent output manually.

agent-skills solves this.

By mapping senior developer workflows directly to structured slash commands, specialist agent personas, and 24 detailed markdown skills, it creates strict quality gates and operating boundaries that AI agents follow autonomously.

How It Works

The agent-skills system orchestrates the agentic software development lifecycle through a multi-phase pipeline of commands and specialist guidelines.

📝

01: Define & Spec (`/spec`)

Establishes the boundaries and design constraints by creating a detailed Product Requirements Document (PRD) before any changes are committed.

02: Interview & Clarify

Uses the `interview-me` skill to conduct a single-question-at-a-time interview, extracting clear requirements when plans are underspecified.

📋

03: Plan Breakdown (`/plan`)

Generates a detailed implementation plan and atomic `task.md` checklist to break down massive architectural changes into manageable tasks.

04: Build & Code (`/build`)

Executes tasks incrementally. `/build auto` runs plans autonomously, pausing on risky steps and committing changes only when verified.

🧪

05: Test & Prove (`/test`)

Drives the Test-Driven Development (TDD) cycle. Enforces that tests are written and passed to guarantee that changes function correctly.

06: Review & Audit (`/review`)

Orchestrates specialist personas (code-reviewer, security-auditor, test-engineer, performance) to audit and refine the codebase before merging.

✂️

07: Code Simplify (`/code-simplify`)

Enforces the `code-simplification` skill to remove unnecessary abstractions, delete dead code, and choose clarity over cleverness.

Aggregates findings, updates metadata versioning files, runs pre-flight scripts, and merges changes safely into production branches.

Get Started

How to Use

01

Install from Marketplace

Add the plugin using the Claude Code shell interface:

/plugin marketplace add addyosmani/agent-skills
/plugin install agent-skills@addy-agent-skills
02

Configure / Local Build

Or clone the repository locally and load the plugin manually:

git clone https://github.com/addyosmani/agent-skills.git
claude --plugin-dir /path/to/agent-skills
03

Activate Commands

Call lifecycle commands directly within your Claude Code workspace session:

/spec  # Design the specification
/plan  # Create task checklist
/build # Implement codebase changes
01

Install Plugin

Install the skills pack directly using the Antigravity command line:

agy plugin install https://github.com/addyosmani/agent-skills.git
02

Run Checks

Invoke skills and lifecycle hooks for active code management:

agy run build-auto
agy run simplify --all
01

Copy Rules

Copy any desired skill definitions from the `skills/` directory into your project's Cursor directory:

mkdir -p .cursor/rules
cp path-to-skills/skills/spec-driven-development/SKILL.md .cursor/rules/spec-driven-development.mdc
02

Interactive Prompting

The rules trigger automatically in Cursor Chat and Composer when related files or design prompts are targeted.

01

Install Skills

Install the skills pack using the Gemini Command Line Interface:

gemini skills install https://github.com/addyosmani/agent-skills.git --path skills
02

Verify Setup

Verify that the native skills are loaded and ready for auto-discovery:

gemini skills list
