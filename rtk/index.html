The Problem

AI Coding Is Burning Through Your Tokens

💸

Massive Token Waste

Every git status, ls, or test run dumps thousands of raw, unfiltered tokens into your LLM context — most of it noise your AI doesn't need.

🧠

Context Window Overflow

LLMs have finite context windows. Bloated command outputs push out valuable conversation history, making your AI assistant forget what you were working on.

⏱️

Slower Responses

More tokens means more processing time and higher costs. A 30-minute coding session can consume ~118,000 tokens on command outputs alone.

RTK solves this.

RTK sits between your AI coding agent and the shell, intelligently filtering and compressing command outputs before they reach your LLM context — cutting token consumption by 60-90% with zero configuration and under 10ms overhead.

Architecture

How It Works

RTK acts as a transparent CLI proxy. When your AI coding agent runs a command, RTK intercepts it, executes the real command, then applies four intelligent filtering strategies to compress the output before it reaches the LLM.

Without RTK Claude / Copilot → git status → ~2,000 tokens (raw)
With RTK Claude / Copilot → RTK proxy → git status → ~200 tokens ✓
🎯

Command Interception

RTK hooks into your AI agent via shell hooks or plugin APIs. When the agent runs a command, it's automatically rewritten to pass through RTK — e.g. git status becomes rtk git status.

🔍

Command Router

RTK's routing engine identifies the command type (git, test runner, linter, file operation) and selects the appropriate filter pipeline from 100+ supported command handlers.

🧹

Smart Filtering

Strips noise — comments, whitespace, boilerplate headers, ANSI color codes, and decorative output that humans need but LLMs don't. Keeps only semantically meaningful content.

📦

Grouping & Aggregation

Similar items are intelligently grouped — files by directory, errors by type, test results by status. Instead of listing 50 files individually, you see directory summaries.

✂️

Truncation

Keeps the most relevant context and cuts redundancy. Long stack traces become concise error summaries. Verbose test output shows only failures, not the hundreds of passing tests.

🔄

Deduplication

Collapses repeated log lines with counts. Instead of showing the same warning 200 times, RTK outputs it once with ×200 — dramatically reducing token count.

Get Started

How to Use

01

Install

Install via Homebrew (recommended), curl, or cargo.

# Homebrew (recommended)
$ brew install rtk

# Quick install (Linux/macOS)
$ curl -fsSL https://raw.githubusercontent.com/rtk-ai/rtk/refs/heads/master/install.sh | sh

# Cargo
$ cargo install --git https://github.com/rtk-ai/rtk
02

Initialize for Your AI Agent

RTK supports Claude Code, Copilot, Gemini CLI, Cursor, Windsurf, Cline, and more.

$ rtk init -g                     # Claude Code / Copilot
$ rtk init -g --gemini            # Gemini CLI
$ rtk init -g --agent cursor      # Cursor
$ rtk init -g --agent windsurf    # Windsurf
$ rtk init --agent cline          # Cline / Roo Code
$ rtk init -g --agent antigravity # Antigravity
03

Use It

Restart your AI tool and start coding. Commands are automatically rewritten to pass through RTK.

# Commands are auto-rewritten by the hook:
$ git status      # → rtk git status (compact output)
$ git diff         # → rtk git diff (condensed diff)
$ cargo test       # → rtk cargo test (failures only, -90%)

# Or call RTK directly:
$ rtk ls .         # Token-optimized directory tree
$ rtk read file.rs # Smart file reading
$ rtk grep "pattern" . # Grouped search results
04

Track Your Savings

Monitor real-time token savings and cost reduction with built-in analytics.

$ rtk gain         # Show token savings stats
$ rtk --version    # Verify installation
