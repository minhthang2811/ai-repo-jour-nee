The Problem

AI Context Windows Are Cluttered

😤

Token Limits Maxed Out

Feeding raw codebases, PDFs, and schemas to AI tools consumes massive context windows and racks up high costs.

🔧

Disconnected Silos

Your database schema, API documentation, and UI code live in different places, making it impossible to query their relationships effectively.

⏱️

Lost Context Between Sessions

Starting a new chat means starting from scratch. You lose the deep context the AI built up, forcing you to constantly re-explain your system.

graphify solves this.

By building a persistent knowledge graph, Graphify maps the relationships across all your files just once. It uses Claude Vision to extract true concepts from anything—code, whiteboard photos, or papers—giving your AI tools a structured 'god-view' of your project without the token bloat.

Get Started

How to Use

01

Install

$ pip install graphifyy && graphify install
02

Configure

# Add to ~/.claude/CLAUDE.md
- **graphify** (`~/.claude/skills/graphify/SKILL.md`) - any input to knowledge graph. Trigger: `/graphify`
When the user types `/graphify`, invoke the Skill tool with `skill: "graphify"` before doing anything else.
03

Run

$ /graphify .
